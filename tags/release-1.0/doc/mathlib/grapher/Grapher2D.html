InheritanceGrapher2D fl.core.UIComponent
Implementsfl.managers.IFocusManagerComponent

The Grapher2D component presents a Cartesian plane and graphs thereon. The addFnGraph() and addPtGraph() methods can be used to add persistent graphs of functions and points that update with changes of the graph's viewing window.

tickStyleproperty 
tickStyle:String  [read-write]

The style of tickmarks to draw. This can be one of three values:

  • Grapher2D.TICKSTYLE_NONE: Don't draw tickmarks.
  • Grapher2D.TICKSTYLE_TICKMARKS: Draw small tickmarks originating from the axes.
  • Grapher2D.TICKSTYLE_GRID: Draw a light grid spaced as tickmarks would be.
The spacing of tickmarks is controlled by the tickDx and tickDy properties. Note that tickmarks will not be drawn if they would be less than 1 pixel apart.

The default value is Grapher2D.TICKSTYLE_TICKMARKS.

This property can be used as the source for data binding.

public function zoom(factor:Number, graphx:Number, graphy:Number, zoomTicks:Boolean = true):void

Zoom the graph window by the specified factor, centering about the given graph coordinate.

Parameters

Constant detail
ABS_MIN_RANGEconstant
public static const ABS_MIN_RANGE:Number = 1e-12

The absolute minimum x- and y-range. This is an arbitrary limit obtained by experimentation—in theory there should be no limit, but if we step too low, Flash starts to have serious issues with the sort of numbers we'd have to deal with (it slows down and eventually hangs). The methods for adjusting the view and zooming check this value and throw a RangeError if a change would go below this value. The whole situation is unfortunate but unavoidable for the time being.
